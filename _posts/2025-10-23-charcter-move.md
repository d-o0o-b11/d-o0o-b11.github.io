## 개요

<!-- 캐릭터 UI -->
저희 UI는 오빗이라는 캐릭터가 이미지로 들어가는 것이 아닌 움직이도록 기획되었습니다.

정적인 이미지가 아닌 살아 숨 쉬는 듯한 캐릭터를 통해 사용자들에게 더 친근하고 생동감 있는 경험을 제공하고 싶었습니다.
오빗은 우주를 떠다니며 사용자의 목표 달성을 응원하는 마스코트 캐릭터로 부드러운 움직임이 핵심이었습니다.


## 첫 번째 시도: Lottie JSON

### 초기 계획

하지만 이전 글에도 얘기했다싶이 그라데이션은 JSON이 지원하지 않습니다..

디자이너분께서 작업하신 애니메이션을 Lottie로 export하면 쉽게 적용할 수 있을 것이라 생각했습니다.
실제로 별 애니메이션은 Lottie로 잘 구현되었기 때문에 캐릭터도 같은 방식으로 가능할 것이라 기대했습니다.

### Lottie의 한계 재발견

하지만 오빗 캐릭터는 단순한 별과 달랐습니다
- 복잡한 그라데이션으로 이루어진 몸체
- 빛나는 효과
- 부드러운 색상 전환

이런 요소들이 Lottie로 export되면 색상이 깨지거나 그라데이션이 단색으로 변하는 문제가 발생했습니다.


## 두 번째 시도: MP4 비디오

### 비디오 포맷 선택

그래서, mp4로 디자이너분께서 만들어주신다고 하셨습니다!

비디오로 렌더링하면 디자인 의도가 100% 반영될 수 있습니다.
색상, 그라데이션, 효과 모두 완벽하게 보존됩니다.

### 새로운 문제 발견

하지만 여기서 발생한 문제점은 mp4는 투명 배경을 만들지 못한다는 것이 였습니다...

**비디오 포맷별 투명도 지원:**
- MP4 (H.264): 투명도 미지원
- WebM (VP9): 투명도 지원하지만 iOS Safari에서 호환성 문제
- MOV (ProRes 4444): 투명도 지원하지만 용량이 너무 큼 (50MB 이상)
- GIF: 투명도 지원하지만 화질 저하, 용량 큼

모든 플랫폼에서 완벽하게 작동하는 투명 배경 비디오 포맷은 없었습니다.


## 좌절과 고민

### 딜레마

투명 배경이 안되면 뒷배경인 그라데이션과 별 부분을 제외해야 하는 걸까,,라는 생각을 계속 했습니다.

어떻게 해야 모든 요소들을 가져갈 수 있을까에 대해 고민을 했습니다.

디자이너분과 저는 회의를 거듭하며 여러 방안을 논의했습니다.
각 방안마다 장단점이 명확했고 무언가를 포기해야 하는 상황이었습니다.

### 검토한 해결 방안들

## 방안 1: 배경까지 포함한 MP4

배경까지 모두 포함된 mp4를 적용하는 방안이 나왔습니다.
하지만, mp4 용량이 커지는 것은 혹시 모를 렉을 유발할 것 같아서 최후의 방법으로 미루었습니다.

**장점:**
- 구현이 가장 간단함
- 디자인 의도 100% 반영
- 모든 플랫폼에서 호환성 보장

**단점:**
- 파일 용량
- 초기 로딩 시간 증가
- 저사양 기기에서 성능 이슈 가능성

네트워크 환경에 따라 사용자 경험이 크게 달라질 수 있어 보류했습니다.

## 방안 2: 배경 요소 제거

움직이는 캐릭터만 가지고 가고 그 외의 뒷배경들을 지우는 방안이 나왔습니다.
이 방안은 디자이너분과 저 둘다 원하지 않는 방안이였기에 기각되었습니다.

**장점:**
- MP4로 캐릭터만 구현 가능
- 파일 용량 최소화

**단점:**
- 우주 컨셉의 핵심인 배경을 포기해야 함
- 밋밋한 UI
- 디자인 의도와 완전히 다른 결과물

디자이너분께서 공들여 만드신 우주 배경을 포기할 수 없었습니다.
그라데이션과 별들이 만들어내는 분위기가 앱의 정체성이기 때문입니다.

## 방안 3: 레이어 분리 구현

움직이는 캐릭터를 최대한 뒷배경을 잘라서 mp4로 만듭니다.
그래서 배경 깔고 그 위에 캐릭터 mp4를 깐 다음에
그 위에 텍스트랑 그라데이션, 별들을 올려보았습니다!

<!-- 움직이는 UI -->

이질감 없이 디자이너분이 작업하신 그대로 화면에 나올 수 있게 되었습니다!!

**레이어 구조:**
```
Layer 1 (최하단): 베이스 우주 배경
Layer 2: CSS 그라데이션
Layer 3: 캐릭터 MP4
Layer 4: Lottie 별 애니메이션
Layer 5: 텍스트 및 UI 요소
```
