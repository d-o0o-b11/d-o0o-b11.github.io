## 개요

OrbitMe로 리브랜딩하면서 UI도 전면적으로 개편하기로 했습니다.
하지만 아직 **이전 miniMe의 흔적**이 남아있는 화면이 있었습니다.

바로 카테고리 추가 화면이었습니다!

<br/>

## Before: miniMe 스타일

![Image](https://github.com/user-attachments/assets/db943cb1-75c2-4c88-8b9f-59d08e5dd738?raw=true)

**이전 UI의 문제점**
- 만다라트 그리드 형태로 OrbitMe 컨셉과 맞지 않음
- 이모지를 어떻게 추가하는지 직관적이지 않음
- 우주 테마와 어울리지 않는 디자인

베타테스트에서도 많은 분들이 혼란스러워하셨습니다:
```
"이모지를 어떻게 넣어요?"
"이 칸을 눌러야 하는 건가요?"
"다음 단계가 뭐죠?"
```

<br/>

## After: OrbitMe 스타일

이 화면을 **오빗미에 어울리는 화면**으로 수정했습니다!

![Image](https://github.com/user-attachments/assets/7671d629-f76b-4353-9fda-9790f76dff49?raw=true)

**개선된 점:**
- 우주 배경과 조화로운 디자인
- 명확한 입력 필드 구분
- 이모지 추가 버튼이 눈에 띔
- OrbitMe 브랜드 아이덴티티 반영

<br/>

## UX 개선: 편집 아이콘 추가

대부분의 유저들이 **이모지를 추가해야하는 부분**을 어려워했습니다.

베타테스터 12명 중 8명이 이모지 추가 방법을 몰라서 헤맸고,
"여기를 눌러야 한다는 걸 어떻게 알아요?"라는 피드백을 받았습니다.

### 해결 방법

어떻게 하면 이모지를 추가할 수 있을까에 대해서 고민을 하다가
**이모지 추가하는 칸 옆에 작게 편집 아이콘**을 추가해보았습니다!


**효과:**
- 사용자가 편집 가능하다는 것을 인지
- 클릭 가능한 영역임을 시각적으로 표현

작은 변화지만 사용자 경험에 큰 차이를 만들었습니다.

<br/>

## 마무리

작은 UI 개선이지만, 사용자 경험에는 큰 영향을 미쳤습니다.

앞으로도 사용자 피드백을 적극 반영하여
더 직관적이고 사용하기 편한 앱을 만들어가겠습니다.
