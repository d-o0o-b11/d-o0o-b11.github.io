## 개요

OrbitMe로 리브랜딩을 진행하면서 앱 전반의 UI 역시 새로운 방향에 맞춰 정리하기로 했습니다.

대부분의 화면은 이미 OrbitMe 컨셉에 맞게 개선되어 있었지만
여전히 이전 miniMe의 흔적이 남아 있는 화면이 하나 있었습니다.
바로 카테고리 추가 화면이었습니다.

이 화면은 앱의 핵심 흐름 중 하나였기 때문에
UI와 UX 모두에서 더 이상 미룰 수 없는 개선 대상이었습니다.

<br/>

## Before: miniMe 스타일

![Image](https://github.com/user-attachments/assets/db943cb1-75c2-4c88-8b9f-59d08e5dd738?raw=true)

기존 카테고리 추가 화면은 miniMe 시절의 구조를 그대로 유지하고 있었습니다.

### 기존 UI의 문제점

- 만다라트 그리드 형태로 OrbitMe의 우주 컨셉과 어울리지 않음
- 이모지 추가 방식이 직관적이지 않음
- 입력 흐름이 명확하게 드러나지 않음

베타테스트 과정에서도 이 화면에서 사용자가 가장 많이 멈춰 서는 모습을 확인할 수 있었습니다.
이는 기능의 문제가 아니라 표현 방식의 문제라고 판단했습니다.

<br/>

## After: OrbitMe 스타일

카테고리 추가 화면을
OrbitMe의 브랜드 방향성과 시각적 언어에 맞게 다시 구성했습니다.

![Image](https://github.com/user-attachments/assets/7671d629-f76b-4353-9fda-9790f76dff49?raw=true)

### 개선된 방향

- 우주 배경과 조화를 이루는 레이아웃
- 입력 영역의 역할이 명확하게 구분된 구조
- 이모지 선택 영역을 눈에 띄게 배치
- OrbitMe의 전체적인 톤앤매너와 일관성 유지

단순히 예쁘게 만드는 것이 아니라 다음 행동이 자연스럽게 보이도록 하는 것에 초점을 맞췄습니다.

<br/>

## UX 개선: 편집 아이콘 추가

베타테스트 결과, 사용자들이 가장 어려워했던 부분은
이모지를 추가해야 하는 위치를 인지하지 못한다는 점이었습니다.

베타테스터 12명 중 다수가
해당 영역을 ‘편집 가능한 요소’로 인식하지 못했습니다.

### 개선 방향

문제의 핵심은 기능이 아니라 신호가 부족하다는 점이었습니다.
이에 따라 이모지 영역 옆에 작은 편집 아이콘을 추가했습니다.

#### 적용 결과

- 사용자가 해당 영역이 편집 가능하다는 점을 인지
- 클릭 가능한 영역임을 시각적으로 명확히 전달
- 별도의 설명 없이도 자연스러운 행동 유도

아주 작은 변화였지만,
사용자가 화면을 해석하는 방식에는 분명한 차이가 생겼습니다.

<br/>

## 마무리하며

카테고리 UI 업데이트는 단순한 디자인 수정이 아니라
OrbitMe의 방향성을 다시 한번 정리하는 작업이었습니다.

작은 UI 요소 하나가
사용자의 행동과 이해도를 크게 바꿀 수 있다는 점을 다시 느꼈습니다.

앞으로도 사용자 피드백과 실제 사용 흐름을 기준으
더 직관적이고 일관된 경험을 만들어가려고 합니다.
