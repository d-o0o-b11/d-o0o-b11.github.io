## 개요

스레드에 글을 올리다보면 종종 달리는 댓글들이 있습니다.

**iPhone 사용자들의 요청**
```
"앱 스토어에서는 설치 못하나요?"
"iOS에서는 설치 못하나요?"
"언제 아이폰에서 쓸 수 있나요?"
```

생각보다 많은 분들이 iOS 버전을 기다리고 계셨습니다.

<br/>

## 원래 계획

원래 저희 계획은 **Android로 100명의 유저를 먼저 모으는 것**이었습니다.

**계획한 이유**
1. 저희가 가려는 방향이 옳은지 확신을 얻고
2. 사용자 피드백을 충분히 받고
3. 안정화를 완료한 뒤에
4. 앱 스토어에 등록

### 망설였던 이유

앱 개발자 계정 비용이 사이드 프로젝트로 하기에는 다소 부담스러웠습니다.

**비용 비교:**
- Google Play: $25 (1회)
- Apple Developer: $129 (연간)

연간 비용이라는 점이 부담되었습니다.

<br/>

## 계획 변경

하지만 스레드에 반응도 좋은 편이라고 생각이 들었고
iOS 사용자들의 요청도 계속 들어왔습니다.

**결정**
바로 앱 스토어에 배포할 준비를 하기로 했습니다!

<br/>

## iOS 심사 준비

### Apple의 엄격한 심사

iOS는 Android보다 심사가 더 엄격하다는 얘기를 종종 들었습니다.

**Apple 심사 가이드라인**
- 앱 완성도
- 개인정보 처리
- 디자인 가이드라인
- 기능 완성도
- **소셜 로그인 정책**

### Apple 로그인 필수

그 중에서 **Apple 로그인**은 현재 **소셜 로그인이 있는 앱에는 필수**라고 들었습니다!

**Apple의 규정**
> 다른 소셜 로그인(Google, Kakao 등)을 제공하는 앱은
> 반드시 Sign in with Apple도 제공해야 합니다.

**현재 제공 중인 로그인:**
- Google 로그인 ✅
- Kakao 로그인 ✅
- Apple 로그인 ❌ ← 추가 필요!

## Apple Developer 계정 구입

### 결심

실제 기기에서 테스트하기 위해서는
**Apple Developer 계정**이 필요했습니다.

$129... 부담스러운 금액이었지만
사용자들의 요청을 생각하며 결제 버튼을 눌렀습니다.

### 계정 승인 대기

**Apple Developer 가입 과정**
1. 개인정보 입력
2. 결제 ($129)
3. 신원 확인
4. 승인 대기 (약 48시간)

이틀을 기다려 드디어 승인을 받았습니다!
저는 금요일 저녁에 결제를 해서 다소 오래 걸린 편인 것 같습니다.

## 배운 점

### 1. 계획은 유연하게

**원래 계획:**
Android 먼저 → 100명 모집 → iOS 출시

**실제:**
Android 출시 → 즉시 iOS 준비

사용자 요청에 빠르게 반응하는 것이 더 중요했습니다.

### 2. 비용보다 가치

$129가 부담스러웠지만
iOS 사용자들에게 서비스를 제공할 수 있다는 것이
더 큰 가치였습니다.

### 3. Apple의 까다로운 심사

Android보다 확실히 준비할 것이 많습니다.
하지만 그만큼 완성도를 높일 수 있는 기회이기도 합니다.
