## 개요

드디어 베타테스트를 시작할 시간이 왔습니다.
Google Play 베타테스트를 위해서는 최소 12명의 테스터가 필요합니다.

갤럭시 유저 12명 모으기가 쉽지가 않더군요,,


## 테스터 모집의 어려움

### 개발자 vs 비개발자

앱을 설치해달라고 부탁하는 것이 개발자들은 이 상황을 알고 있기에 흔쾌히들 수락해 주시지만,
개발 직군이 아닌 다른 직군의 지인들은 조금 불편해하는게 느껴졌습니다.

**개발자 지인들의 반응**
- "오! 뭐 만들었어? 당연히 깔아줘야지"
- "베타테스트 필요하구나, 괜찮아 바로 설치할게"
- "피드백 바로바로 줄게!"

**비개발자 지인들의 반응**
- "음... 베타테스트가 뭐예요?"
- "괜찮은 건가요? 안전한 거죠?"
- "제 폰에 깔아도 되나요...?"

### 보안에 대한 우려

워낙 요즘 보안 관련해서 이슈도 많다보니, 어느정도 이해는 갑니다 ㅠㅠ

**사람들이 걱정하는 것들**
1. "개인정보가 유출되지 않을까?"
2. "이상한 권한 요구하는 거 아니야?"
3. "폰이 해킹되는 건 아니겠지?"
4. "유료 결제가 발생하는 건 아니죠?"

이런 우려들을 하나하나 설명하고 안심시키는 것도 쉽지 않았습니다.


## 발견된 이슈들

12명을 모으고 사용자들이 바로바로 피드백을 주셨습니다.

### 이슈 1: 글씨가 깨진다

폰트 크기가 보통 보다 커지면 안보이는 상황이 발생했습니다.

**문제 상황:**
- 사용자 폰의 시스템 폰트 크기: "크게" 또는 "매우 크게" 설정
- 앱 내 텍스트가 컨테이너를 벗어남
- 일부 텍스트가 잘려서 안 보임

**재현 방법:**
```
1. 폰 설정 > 디스플레이 > 글꼴 크기 > "매우 크게" 선택
2. 앱 실행
3. 메인 화면의 목표 제목 확인
4. 텍스트가 영역을 벗어남
```

### 이슈 2: 카테고리 추가하는 방법을 모른다

카테고리 추가하는 화면에서 이모지를 넣어야 한다는 것을 사용자들이 파악하지 못하는 상황입니다.

**문제 상황:**
```
사용자: "카테고리 추가 버튼을 눌렀는데..."
사용자: "이모지만 있고 다음에 뭘 해야 하는지 모르겠어요"
사용자: "이모지를 선택하면 되는 건가요?"
```

### 이슈 3: 무슨 앱인지 모른다

메인 페이지에서 카테고리 추가하는 것까지는 했지만, 그 다음엔 어떤걸 해야하는지 방황을 하였습니다.
이 앱에 대한 설명이 부족했던 것 같습니다.

**사용자 여정 분석:**

**현재:**
```
앱 실행 → 빈 화면 → "+" 버튼 발견 → 카테고리 추가 → "이제 뭐하지?"
```

**사용자 피드백:**
- "이게 목표 관리 앱인가요?"
- "카테고리를 왜 만드는 거죠?"
- "다른 앱들처럼 튜토리얼 같은 게 있으면 좋겠어요"

**문제점:**
1. 온보딩 부족
2. 앱의 목적 설명 부족
3. 각 기능의 의미 불명확
4. 사용 흐름 가이드 없음

## 배운 점

### 1. 실제 사용자 테스트의 중요성

**개발자 테스트 vs 실제 사용자 테스트:**

| 구분 | 개발자 | 실제 사용자 |
|------|--------|------------|
| 앱 이해도 | 100% | 0% |
| 사용 방식 | 정해진 시나리오 | 자유롭게 |
| 발견하는 이슈 | 기능적 버그 | UX 문제 |

아무리 꼼꼼히 테스트해도 실제 사용자의 시선으로 보면 전혀 다른 문제가 보입니다.

### 2. 당연한 것은 없다

**개발자가 당연하다고 생각한 것**
- "이모지 아이콘을 누르면 되는 건 당연하지"
- "이 앱이 목표 관리 앱인 건 뻔한데"
- "플러스 버튼은 추가 버튼인 거 다 알잖아"

**사용자 입장:**
- "이거 어떻게 사용하는 건데?"
- "이게 뭐하는 앱이지?"
- "이 버튼을 누르면 뭐가 되는 거야?"

우리에게 당연한 것이 사용자에게는 전혀 당연하지 않습니다.

### 3. 온보딩의 중요성

사용자는 앱을 처음 봤을 때 3초 안에 판단합니다:
- "이 앱이 뭐하는 건지 알겠는가?"
- "내게 필요한 앱인가?"
- "사용법을 알 수 있는가?"

온보딩은 선택이 아닌 필수라는 것을 느꼈습니다..
