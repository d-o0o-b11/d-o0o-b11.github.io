## 개요

드디어 베타테스트를 시작할 시점이 되었습니다.
Google Play 베타테스트를 진행하기 위해서는 최소 12명의 테스터가 필요했고
이를 충족하는 것이 첫 번째 관문이었습니다..

안드로이드 사용자 12명을 모으는 일이 생각보다 쉽지 않았습니다.
이 단계부터 이미 개발과는 또 다른 종류의 현실적인 벽을 느끼게 되었습니다.

<br/>

## 테스터 모집 과정에서 느낀 점

### 개발자와 비개발자의 인식 차이

테스터를 모집하는 과정에서, 개발자 지인과 비개발자 지인 간의 반응 차이가 분명하게 드러났습니다.

개발자들은 베타테스트라는 개념과 목적을 이미 알고 있었기 때문에 비교적 부담 없이 참여해주었습니다.
반면, 개발 직군이 아닌 지인들에게는 베타테스트 자체가 생소했고 앱 설치에 대한 심리적 장벽이 존재했습니다.



### 보안에 대한 우려

특히 눈에 띄었던 부분은 보안에 대한 걱정이었습니다.
요즘 개인정보 유출이나 악성 앱 관련 이슈가 많다 보니, 이러한 반응은 충분히 이해할 수 있었습니다.

많은 사람들이 다음과 같은 부분을 우려했습니다.

- 개인정보 수집 여부
- 불필요한 권한 요청
- 기기 보안 문제
- 의도치 않은 결제 발생 가능성

베타테스트 참여를 설득하는 과정에서,
기능 설명뿐만 아니라 신뢰를 설명하는 과정이 필요하다는 점을 실감하게 되었습니다.

<br/>

## 베타테스트에서 발견된 주요 이슈들

테스터 12명을 모은 후, 예상보다 빠르게 피드백이 모이기 시작했습니다.
놀랍게도 단 하루 만에 치명적인 문제들이 연이어 드러났습니다.


### 이슈 1: 시스템 폰트 크기 증가 시 텍스트 깨짐

시스템 폰트 크기를 크게 설정한 사용자 환경에서
앱 내 텍스트가 컨테이너를 벗어나거나 잘리는 문제가 발생했습니다.

#### 문제 상황

- 시스템 글꼴 크기: ‘크게’ 또는 ‘매우 크게’
- 텍스트가 영역을 초과하여 일부 내용이 보이지 않음

이 문제는 개발 환경에서는 쉽게 드러나지 않았지만,
실제 사용자 환경에서는 충분히 재현 가능한 이슈였습니다.

<br/>


### 이슈 2: 카테고리 추가 방식의 비직관성

카테고리 추가 화면에서 이모지를 선택해야 다음 단계로 진행할 수 있다는 점을
다수의 사용자가 인지하지 못했습니다.

UI 상에서는 선택 요소가 존재했지만,
사용자가 이를 ‘행동해야 할 요소’로 인식하기에는 안내가 부족했습니다.

> “보여주는 것”과 “이해시키는 것”은 다르다는 점을 다시 한번 느끼게 되었습니다.
{: .prompt-info }

<br/>

### 이슈 3: 앱의 목적이 명확하지 않음

카테고리를 추가한 이후,
다음 행동을 무엇을 해야 할지 몰라 흐름이 끊기는 현상이 발생했습니다.

이는 기능의 문제가 아니라 앱의 방향성과 사용 목적이 충분히 전달되지 않았기 때문이었습니다.

문제의 핵심은 다음과 같았습니다.

- 온보딩 부재
- 앱의 핵심 가치 설명 부족
- 기능 간 흐름 가이드 없음

이 이슈는 단순 UI 개선이 아닌
앱 전체 경험을 다시 정의해야 하는 수준의 문제였습니다.

<br/>

### 이슈 4: StatusBar에 UI가 가려지는 문제

버전 업 이후 일부 화면에서 상단 UI가 StatusBar 영역에 가려지는 현상이 발생했습니다.

모든 화면에 영향을 줄 수 있는 문제였기에
우선순위를 가장 높게 두고 대응해야 했습니다.

<br/>

## 배운 점

### 1. 실제 사용자 테스트의 중요성

아무리 내부 테스트를 반복해도
실제 사용자의 환경과 시선에서는 전혀 다른 문제가 드러납니다.

기능은 정상적으로 동작하더라도
UX는 언제든 실패할 수 있다는 사실을 체감했습니다.

### 2. 개발자에게 당연한 것은 사용자에게 당연하지 않다

개발 과정에서 자연스럽게 여겨졌던 요소들 대부분이 사용자에게는 설명되지 않은 상태였습니다.

- 버튼의 의미
- 행동의 결과
- 앱의 목적

이 모든 것은 명시적으로 전달되어야 했습니다.

### 3. 온보딩은 선택이 아니라 필수다

사용자는 앱을 켜는 순간부터 판단을 시작합니다.

- 이 앱은 무엇을 하는가?
- 나에게 어떤 도움이 되는가?
- 어떻게 사용해야 하는가?

이 질문에 빠르게 답하지 못한다면
사용자는 바로 이탈할 수밖에 없습니다.

이번 베타테스트를 통해
온보딩은 기능이 아니라 생존 요소라는 것을 분명히 깨닫게 되었습니다. 허허,,
