## 개요

기존에 배포된 miniMe는 만다라트 UI를 중점으로 가져갔습니다.
하지만, 유저들을 모으기 위해서는 더 특별한 무언가가 필요하다고 생각이 들었습니다.

그래서 우주 컨셉으로 가져가도록 의견을 모았습니다.


## UI 변경의 배경

### 만다라트 UI의 한계

만다라트는 목표 관리 방법론으로서는 훌륭했지만, 시각적으로 차별화된 경험을 제공하기에는 부족함이 있었습니다.
실제로 비슷한 컨셉의 앱들이 이미 많이 존재했고, 우리 앱만의 정체성을 만들어내기 어려웠습니다.

### 우주 컨셉 선택의 이유

최종적으로 우주 컨셉을 선택한 이유는
1. **확장성**: 목표가 커지고 성장하는 모습을 행성이 성장하는 것으로 은유적으로 표현 가능
2. **차별성**: 기존 목표 관리 앱들과 확연히 다른 UI
3. **몰입감**: 우주라는 광활한 공간이 주는 상상력과 가능성
4. **애니메이션 활용**: 행성의 궤도 운동, 별의 반짝임 등 동적인 요소 추가 가능


## 어려운 UI

### 프론트엔드 개발 경험의 부족

사실 저는 프론트를 집중적으로 개발한 경험이 대학교 졸업과제 외에는 없었습니다.
그래서 단순한 UI는 몇번 해보았지만 복잡한 UI(반응형 궤도(?)원형, 그 궤도를 도는 작은 행성들(애니메이션), json 배경 등)는 한 번도 구현해보지 못해서 조금 애를 먹었습니다.

주로 백엔드 개발에 집중했던 저에게 이번 UI 작업은 새로운 도전이었습니다.
CSS 애니메이션, 반응형 레이아웃 등 배워야 할 것들이 산더미처럼 쌓여있었습니다.

### AI를 활용한 개발

하지만 요즘은 AI도 많이 발달되어 있고 잘만 활용하면 만들 수 있을 것 같아서 디자이너분이 작업해 주신 UI 최대한 구현하고자 했습니다.

ChatGPT, Claude, Copilot 등을 적극 활용하며
- CSS 애니메이션 코드 작성
- 반응형 레이아웃 구조 설계
- 성능 최적화 방법

이런 부분들에 대해 질문하고 제안받은 코드를 이해하고 수정하며 구현해 나갔습니다.

## 배운 점과 느낀 점

### 디자인-개발 협업

디자이너분과의 소통이 중요했습니다
- 구현 가능성에 대한 사전 논의
- 디자인 시안과 실제 구현의 차이 조율
- 성능과 비주얼의 균형점 찾기

처음에는 "디자인대로 100% 구현해야 한다"는 강박이 있었지만
실제로는 기술적 제약과 사용자 경험을 고려하여 적절히 조정하는 것이 중요하다는 것을 배웠습니다.

### AI 도구의 올바른 활용

AI는 강력한 도구이지만 맹신해서는 안 됩니다
- AI가 제안한 코드를 이해하고 수정할 수 있어야 함
- 여러 AI 도구의 답변을 비교하고 검증
- 기본기가 탄탄해야 AI를 효과적으로 활용 가능

### 작은 성취의 중요성

오늘은 UI만 작업하는 것을 목표로 하였습니다.

큰 목표를 세우는 것도 좋지만, 하루하루 작은 목표를 설정하고 달성하는 것이 동기부여에 큰 도움이 되었습니다.
"오늘은 궤도만 완성하자", "오늘은 애니메이션만 붙이자" 이런 식으로 접근하니 부담도 덜하고 성취감도 더 자주 느낄 수 있었습니다.
